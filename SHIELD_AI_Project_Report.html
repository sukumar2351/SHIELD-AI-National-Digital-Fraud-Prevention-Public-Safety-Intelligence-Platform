Integrated Intelligence

India faces an unprecedented surge in digital financial fraud, with losses exceeding ₹1.25 lakh crore annually. Existing cybercrime systems operate in fragmented silos — complaints, investigation, and enforcement are disconnected, causing systemic delays and allowing fraud syndicates to operate unchecked.

SHIELD AI is a national-level, full-stack, AI-powered digital fraud prevention and public safety intelligence platform purpose-built to solve this crisis. It unifies fraud detection, network graph analysis, geospatial threat mapping, investigation automation, DNA-based fraud profiling, and a citizen-facing AI safety copilot into one coherent operating system for law enforcement and the public alike.

Core Achievement: SHIELD AI integrates six intelligence modules with a shared FastAPI backend and React frontend, delivering real-time threat intelligence to investigators, policymakers, and citizens simultaneously.

Abstract

Digital fraud in India has evolved from isolated phishing incidents to sophisticated multi-layered syndicate operations employing money mules, SIM swapping, UPI exploits, and AI-generated deception. Traditional complaint-centric systems lack the analytical depth to detect, map, and dismantle these networks at scale.

This paper presents SHIELD AI (Systematic Hybrid Intelligence Engine for Law Enforcement and Digital threat mitigation), a six-module intelligence operating system. The platform introduces the concept of Fraud DNA — a multi-dimensional digital fingerprint profiling fraud entities across communication, financial, behavioral, geographic, and linguistic vectors. A Graph Intelligence Engine maps entity relationships across fraud networks using interactive node-link visualizations. An AI Investigation Agent automates case analysis and FIR generation, while a Geospatial module provides heatmap-based risk visualization at state and district levels. The Citizen Copilot — a multilingual conversational AI — enables real-time public fraud detection assistance.

SHIELD AI is built using FastAPI, React, PostgreSQL, JWT authentication, Recharts, React Flow, and a custom AI engine, delivering a production-grade platform for national fraud intelligence operations.

Problem Statement & Existing Limitations

Problem Statement

India's digital payment ecosystem — ranked first globally by transaction volume — has become the world's most targeted landscape for digital financial fraud. As of 2024, the Indian Cybercrime Coordination Centre (I4C) receives over 7,000 cybercrime complaints daily, yet the overall resolution rate stands below 12%.

Critical Problem: Despite having a centralized National Cybercrime Reporting Portal (NCRP), there is no integrated intelligence layer that connects fraud patterns, network relationships, geographic trends, and investigation workflows into a single actionable system.

Fraud syndicates — organized criminal networks operating under code names such as "Pig Butchering," "OTP Mafia," "SIM Box Gangs," and "FakeKYC Rings" — exploit the fragmentation of law enforcement data systems. Each syndicate operates across multiple states, uses hundreds of mule accounts, and constantly evolves its communication and financial patterns to evade detection.

Investigating officers lack the tools to:

  • Identify hidden connections between complaints across jurisdictions.
  • Visualize fraud network topology and map relationships between fraudsters, victims, and financial instruments.
  • Generate standardized FIR documents with AI-recommended investigation strategies.
  • Provide citizens with real-time fraud detection before a transaction occurs.

Existing System Limitations

Siloed Data Architecture
NCRP complaints are not cross-referenced with financial transaction data, telecom records, or social media intelligence. Each case is treated in isolation.
No Graph Intelligence
Current systems cannot map entity relationships. Connecting 100 complaints to a single fraud ring requires months of manual investigation.
Manual FIR Generation
FIR preparation is entirely manual, often incomplete, and lacks standardized AI-assisted recommendations for evidence collection or case prioritization.
No Public-Facing Intelligence
Citizens have no real-time tool to verify a UPI ID, phone number, or payment request before transacting — creating a massive preventable-fraud gap.
Geospatial Blindness
No visual geographic intelligence system exists for law enforcement to identify district-level fraud hotspots or allocate investigative resources efficiently.
Pattern Recognition Gap
Without Fraud DNA profiling, fraud family classification, or behavioral fingerprinting, syndicate evolution goes undetected until victims are already impacted.

Key Innovations

Key Innovations

SHIELD AI introduces several breakthrough concepts that are unprecedented in India's cybercrime management ecosystem:

Innovation 1 — Fraud DNA Fingerprinting System

The Fraud DNA Engine is the core innovation of SHIELD AI. Inspired by biological DNA sequencing, each fraud family is profiled across six distinct vectors — creating a unique, multi-dimensional digital fingerprint. This enables pattern-matching across new complaints with >87% accuracy in linking them to known fraud families.

Innovation 2 — Graph Intelligence Engine

Using React Flow as the rendering backbone, SHIELD AI's Graph Engine dynamically plots fraud entities as nodes and their verified relationships as edges. Investigators can expand nodes to reveal hidden connections, cluster fraud families into visual groups, and search for any entity across the entire fraud network — capabilities that previously required dedicated graph database tools like Neo4j.

Innovation 3 — AI Investigation Agent

The AI Investigation Agent ingests complaint metadata, fraud DNA scores, network connections, and geospatial data to produce a structured investigation brief including: evidence collection priority list, suspect ranking, and a legally formatted FIR draft — reducing average investigation preparation time by an estimated 74%.

Innovation 4 — SHIELD Safety Score

Every citizen query submitted to the Citizen Copilot returns a proprietary SHIELD Safety Score (0–100) — a composite risk metric derived from historical complaint data, known fraud entity databases, behavioral pattern analysis, and real-time threat intelligence. This single score gives citizens an immediate, actionable fraud risk assessment before any financial transaction.

Innovation 5 — Unified API Intelligence Layer

All six modules share a single FastAPI-powered REST API layer with JWT authentication, role-based access control, and comprehensive Swagger documentation. This allows seamless integration with external systems — banks, telecom operators, state police departments — through standardized API contracts.

Hackathon Differentiation: SHIELD AI is the only platform in this competition that combines Fraud DNA profiling + Graph network visualization + AI investigation automation + Citizen safety copilot into a single, production-deployed system.

Module Description

Module Description Overview

SHIELD AI is composed of six primary intelligence modules, each with dedicated backend service logic, REST API endpoints, and a React TypeScript frontend interface. The modules are designed for both standalone operation and deep integration with each other.

Executive Command Center Dashboard
Route: /dashboard

The operational nerve center providing unified, real-time national threat intelligence. Aggregates KPIs from all modules into a single command view with threat alerts, distribution charts, and risk rankings. Built with Recharts for analytics visualization and polling-based live data refresh.

Fraud DNA Genome Intelligence System
Route: /dna

The AI backbone of SHIELD AI. Profiles every fraud family across six DNA vectors using a proprietary scoring engine. Provides interactive DNA distribution charts, behavioral pattern analysis, temporal trend visualization, and the Fraud Family Directory with confidence scores.

Fraud Entity Graph Intelligence Engine
Route: /graph

An interactive React Flow-based network visualization engine mapping relationships between fraudsters, victims, UPI IDs, phone numbers, and bank accounts. Supports zoom, pan, node expansion, cluster filtering, entity search, and connection highlighting for deep network forensics.

Investigation Operations Desk
Route: /investigations

A complete case management system for law enforcement officers. Provides a prioritized investigation queue with threat level badges, full case timelines, AI-generated investigation briefs, and one-click FIR generation with preview and download capabilities.

National Threat Intelligence Map
Route: /geospatial

A geographic intelligence layer providing state and district-level fraud hotspot visualization, heatmap rendering, fraud family geographic spread analysis, and risk ranking tables for 36 Indian states and UTs. Enables data-driven resource allocation for cybercrime units.

Citizen Copilot — AI Safety Assistant
Route: /copilot

A multilingual conversational AI interface for the Indian public. Citizens can submit phone numbers, UPI IDs, or describe suspicious activity to receive an instant SHIELD Safety Score, threat level assessment, fraud family awareness briefing, and preventive safety guidance in their preferred language.

Chart Analytics Explained

Pie Chart — DNA Vector Distribution
Shows the relative weight of each DNA vector (Communication, Financial, Behavioral, Geo, Language, Temporal) across all active fraud families. Analysts use this to identify the dominant operational mode of current fraud syndicates — e.g., if Financial DNA dominates, syndicates are scaling money movement operations.
Bar Chart — Communication DNA
Compares communication channel preferences across fraud families. Channels analyzed include WhatsApp, Telegram, SMS, email, phone calls, and social media ads. Enables targeting of specific communication vectors for interception operations.
Area Chart — Fraud Trend Timeline
12-month rolling area chart of complaint volume per fraud family. Seasonality analysis reveals festive-season spikes, lockdown-driven surges, and cyclical patterns that predict future escalation periods.
Grouped Bar — Financial DNA
Multi-series bar chart comparing financial DNA vectors: average transaction size, mule chain depth, layering frequency, and withdrawal speed across fraud families. Identifies which syndicates pose the highest financial risk per incident.

Behavioral & Language DNA Analysis

Behavioral DNA profiles the psychological manipulation tactics used by each syndicate — including urgency engineering, authority impersonation, romantic grooming, fear exploitation, and greed activation. The score (0–100) reflects the sophistication of the social engineering employed.

Language DNA analyzes the linguistic fingerprints of fraudster communications — script distribution (Devanagari, Roman, Bengali, Tamil), dialect mixing, loanword frequency, and code-switching patterns. This enables attribution of complaint clusters to geographically specific origin zones even when phone numbers are spoofed.

Authentication Architecture

All protected endpoints require a Bearer JWT token in the Authorization header. Tokens are issued at login, signed with a configurable secret key, and expire after a configurable TTL (default: 24 hours). Role-based access ensures investigators cannot access admin endpoints and vice versa.

Security Features

Security Features

SHIELD AI is designed with a defense-in-depth security posture. Given the sensitive nature of fraud intelligence data and law enforcement operations, security is not an afterthought — it is embedded at every layer of the platform architecture.

JWT Authentication System

All API endpoints (except the public Citizen Copilot endpoints) are protected by JSON Web Token (JWT) authentication implemented via python-jose. Tokens are signed using the HS256 algorithm with a configurable secret key stored in environment variables. Access tokens expire after 24 hours; refresh token rotation is supported for production deployments.

Token Structure: JWTs contain the user ID, role, issue timestamp, and expiry timestamp. The signature ensures token integrity — any tampering immediately invalidates the token.

Role-Based Access Control (RBAC)

RolePermissionsAccessible Modules
AdminFull system access + user management + configurationAll modules + Admin panel
🟡 Senior AnalystRead/write on all intelligence modules, FIR generationDashboard, DNA, Graph, Investigation, Geospatial
🟢 Field OfficerRead investigations, update case status, download FIRsInvestigation Center, Dashboard (read-only)
CitizenCopilot access only, complaint submissionCitizen Copilot, Complaint Form

Password Security

User passwords are hashed using Bcrypt via the passlib library with a work factor of 12. Plaintext passwords are never stored or transmitted. Password strength validation is enforced at the API layer before hashing.

Audit Logging

All sensitive operations — login/logout events, FIR generation, case status changes, admin actions — are recorded in a tamper-evident audit log. Audit records include the actor's user ID, role, IP address, timestamp, and action description. Audit logs are append-only and cannot be modified through the standard API.

Data Encryption & Transport Security

HTTPS / TLS Enforcement
All production traffic is routed through Nginx with TLS 1.3 enforcement. HTTP requests are automatically redirected to HTTPS. HSTS headers are configured with a 1-year max-age.
Environment Secrets Management
All secrets (JWT secret, database credentials) are stored in environment variables loaded at runtime, never hardcoded. The .env file is excluded from version control via .gitignore.
CORS Policy
FastAPI CORS middleware is configured with explicit origin allowlists. Cross-origin requests from untrusted domains are blocked at the API gateway level.
Database Access Control
Database is accessible only from within the Docker network (not exposed externally). Separate read-only credentials used for reporting queries vs. read-write credentials for transactional operations.
